# خطة Hotelbeds Accommodation Mapping

## الهدف

تجهيز تغطية محتوى قريبة من 90% لمنتجات Hotelbeds Accommodation بدون الاعتماد على Availability كأساس للمحتوى الثابت، وبدون استخدام بيانات سعر قديمة عند البحث أو الحجز.

## الموجود حاليًا

- يوجد عميل Hotelbeds Content API مستقل.
- توجد دوال لجلب الفنادق، تفاصيل الفندق، الدول، الوجهات، الغرف، المرافق، التصنيفات، العملات، واللغات.
- توجد طبقة إثراء تربط نتائج Hotelbeds بالصور والوصف والعنوان والمرافق عند توفرها.
- توجد روابط صور Hotelbeds مبنية من مسارات الصور القادمة من Content API.
- يوجد كاش ذاكرة مؤقت للإثراء أثناء التطوير، لكنه ليس تخزينًا دائمًا.

## المطلوب لتغطية 90%

- جلب الدول والوجهات المدعومة من Content API.
- جلب الفنادق على دفعات حسب الوجهات أو الدول، مع احترام حدود الطلبات.
- حفظ `hotelCode` و`destinationCode` و`countryCode` واسم الفندق والوصف والصور والعنوان والإحداثيات والتصنيف.
- حفظ ملخص الغرف والمرافق والتصنيفات واللوحات `boards` والمراجع اللازمة للعرض.
- تحديث المحتوى دوريًا بعملية batch sync بدل استدعاءات لحظية كثيفة.
- استخدام البيانات المخزنة للبحث والاقتراحات والصور والوصف.
- استخدام Availability live فقط للسعر والتوفر والسياسات الحديثة.
- استخدام Check Rate قبل أي حجز لتثبيت السعر والسياسات النهائية.

## ما لا يجب فعله

- لا يتم استخدام Content API لتأكيد حجز.
- لا يتم الاعتماد على سعر مخزن عند الحجز.
- لا يتم تشغيل مزامنة واسعة داخل تجربة العميل.
- لا يتم استخدام Promise.all أو طلبات متوازية عند التعامل مع Hotelbeds أثناء الشهادة.
- لا يتم تجاوز حدود quota أو تنفيذ retries عشوائية.

## خطة التنفيذ المقترحة

1. إنشاء جدول/مجموعة تخزين لمحتوى Hotelbeds Accommodation.
2. تنفيذ sync للوجهات والدول أولًا.
3. تنفيذ sync للفنادق على دفعات صغيرة، مع عداد طلبات وتأخير بين الطلبات.
4. حفظ الصور والأوصاف والمرافق في شكل موحد تستخدمه الواجهة.
5. ربط autocomplete بقاعدة البيانات عند توفرها.
6. إبقاء Availability وCheck Rate live للسعر والتوفر فقط.
7. إضافة تقرير تغطية يقيس عدد الفنادق المخزنة مقابل portfolio المتاح من Hotelbeds.

## حالة الجاهزية الحالية

البنية الأساسية موجودة، لكن تغطية 90% تحتاج batch sync وتخزين دائم وتقرير تغطية. لذلك الحالة الحالية جاهزة للتطوير والاختبار المحدود، وليست جاهزة بعد كـ mapping كامل للإنتاج.
