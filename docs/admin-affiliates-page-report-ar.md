# تقرير صفحة إدارة الأفلييت

تاريخ التقرير: 2026-05-25

تم الاعتماد على الكود الحالي فقط، ولم يتم الاعتماد على README. لم يتم تفعيل Stripe live، ولم يتم استخدام أي API حقيقي للموردين، ولم يتم ربط العمولة بحجز حقيقي أو تغيير منطق الحجز أو الدفع.

## الملفات المضافة

- `app/[locale]/admin/affiliates/page.tsx`

## الملفات المعدلة

- `components/admin/admin-sidebar.tsx`

## رابط صفحة الأفلييت

```text
/ar/admin/affiliates
/en/admin/affiliates
```

مثال محلي:

```text
http://localhost:3000/ar/admin/affiliates
```

## ما الذي يظهر في الصفحة الآن

- ملخص عام لحالة نظام الأفلييت.
- أقسام واضحة لـ:
  - المسوقون / Affiliates
  - روابط الإحالة
  - أكواد الخصم
  - الإحالات
  - العمولات
  - حالة الدفع / Payouts لاحقًا
- Empty State واضح:
  - "لا توجد بيانات أفلييت حتى الآن"
  - توضيح أن البيانات ستُربط بالحجوزات المؤكدة لاحقًا.
- شرح داخل الصفحة بأن النظام حاليًا بنية أولية فقط.
- عرض APIs والنماذج الموجودة في الكود بدون استدعائها من الصفحة.

## ما الذي ينتظر الربط لاحقًا

- ربط بيانات المسوقين فعليًا بقاعدة البيانات من واجهة أدمن آمنة.
- ربط الإحالة بالحجز الحقيقي بعد اكتمال مسار الحجز.
- احتساب العمولة فقط عند وصول الحجز إلى `supplier_booking_confirmed`.
- تطبيق أكواد الخصم على checkout بعد اعتماد قواعد الخصم.
- بناء payout حقيقي بعد اعتماد العمولات ومراجعة الأمان.
- إضافة صلاحيات ومصادقة واضحة قبل فتح إدارة الأفلييت للإنتاج.

## نتائج الفحوصات

تم تشغيل:

```text
npm.cmd run lint
npm.cmd run typecheck
npm.cmd run build
```

النتائج:

- `lint`: ناجح.
- `typecheck`: ناجح.
- `build`: ناجح، وظهر المسار الجديد `/[locale]/admin/affiliates`.

تحذير غير مانع ظهر أثناء البناء:

```text
baseline-browser-mapping data is over two months old
```

هذا التحذير لا يمنع التشغيل أو البناء.
